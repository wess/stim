# planning

Three commands for thinking before coding.

| Command | What it does |
|---------|--------------|
| `/spec` | Walks you through the questions that turn a fuzzy idea into a SPEC.md. Focuses on what's out of scope as much as what's in. |
| `/breakdown` | Takes a spec or feature description and produces TASKS.md — ordered, testable, each under two hours. |
| `/scope` | Runs a reality check on the scope of what you're about to build. Forces you to name cuts and identify where the risk lives. |

## Install

```bash
stim add github/wess/stim/packages/planning
```

Then use `/spec`, `/breakdown`, `/scope` in Claude Code.
